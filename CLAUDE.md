# Pokenese — Project Guide

## What Is This

Pokenese is a Wordle-style daily game that teaches Mandarin Chinese through Pokémon names. Players see a Pokémon's Chinese name and guess the English name, receiving hints on wrong guesses.

## Key Documents

- `README.md` — High-level goals and feature overview (source of truth for intent)
- `SPEC.md` — Full technical and functional specification (architecture, APIs, data models, game logic)
- `VISUAL_DESIGN_SPEC.md` — Complete UI/UX specification (design tokens, screen layouts, components)

## Planned Stack

- **Frontend:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, Framer Motion, Zustand, TanStack Query
- **Backend:** Python FastAPI, PostgreSQL, SQLAlchemy async, JWT auth
- **Data:** TypeScript constants in `frontend/src/data/pokemon.ts` (generated by scraping script)

## Notes

- Mobile-first design
- Anonymous users supported via localStorage; accounts optional
- Pokémon data scraped and stored as TypeScript constants (not fetched at runtime from external APIs)
- Daily challenges pre-seeded in DB; same 3 Pokémon for all users worldwide each day
- Challenge mode resets on complete failure (score 0 on a challenge)
